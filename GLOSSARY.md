# Glossary

- `semantic kernel`: the canonical typed world model for software meaning
- `internal builder`: a subsystem that materializes semantic meaning into runnable software artifacts on our own substrate
- `proof harness`: the test and replay system that verifies generated systems
- `RALPH loop`: the bash-supervised control loop around the semantic loop
- `semantic loop`: observe, infer, canonicalize, build, prove, compare, harvest
- `provenance`: the record of where a semantic fact or change came from
- `benchmark app`: a canonical application family used to measure progress
- `super-user`: a user who gains software-building power from the product without ceasing to be a domain expert
- `agent swarm`: the coordinated set of specialized agents that run Ralph stages and emit typed artifacts
- `function-call artifact`: a structured record of which tool-like function was invoked, by whom, with what payload, and what it returned
