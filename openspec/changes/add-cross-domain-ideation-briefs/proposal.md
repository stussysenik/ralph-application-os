## Why

Ralph could already take prompts into interviews and workflow-heavy drafts, but the intake path still treated too many ideas as if they were applications on the current runtime wedge. That is misleading for categories like compilers, renderers, kernels, protocols, and scientific systems.

## What Changes

- add a deterministic cross-domain ideation brief before deeper synthesis
- classify prompts into software categories with explicit execution-mode reporting
- extend interview generation with category-specific question packs
- persist a generated architecture outline for the ideation result
- expose the ideation surface through a first-class `ralph:ideate` operator command

## Impact

- Ralph can now take a broader class of software ideas seriously without promising the wrong kind of execution
- operators get category-specific proof and architecture pressure earlier
- the repo moves closer to a real universal ideation front door while staying honest about runtime limits
