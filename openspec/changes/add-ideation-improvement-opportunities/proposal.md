## Why

Ralph can already improve ideas at the draft handoff layer, but the universal `ralph:ideate` entrypoint still behaves mostly as a classifier and interview launcher. That leaves product-shaping value too late in the flow.

## What Changes

- add deterministic `improvementOpportunities` to ideation briefs
- render those opportunities in the ideation report
- keep the suggestions category-aware so broader software prompts still get useful pressure before draft synthesis

## Impact

- ideation briefs become more useful before interview answers exist
- Ralph improves ideas earlier instead of waiting for a full semantic draft
- unsupported software categories still get actionable product and architecture pressure
